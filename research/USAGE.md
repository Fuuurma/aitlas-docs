# Aitlas Core Template - Usage Guide

## Quick Start

```bash
# 1. Clone the template
git clone https://github.com/Fuuurma/aitlas-core-template.git my-product
cd my-product

# 2. Remove template git history
rm -rf .git
git init

# 3. Install dependencies
bun install

# 4. Run setup (interactive)
bun run setup

# 5. Set up database
bun run db:generate
bun run db:migrate

# 6. Start developing
bun run dev
```

## What Setup Does

The `bun run setup` command:

1. **Prompts for configuration:**
   - Product name (e.g., `f.decloy`)
   - Product description
   - Product slug
   - Author/Organization
   - Repository URL

2. **Updates all source files** with your configuration

3. **Creates `.env.local`** with generated secrets:
   - `BYOK_ENCRYPTION_KEY` - AES-256-GCM key
   - `NEXTAUTH_SECRET` - NextAuth.js secret

4. **Creates `aitlas.config.json`** with saved configuration

## Furma DNA

Every Aitlas product includes these core utilities:

| File | Purpose |
|------|---------|
| `lib/encryption.ts` | AES-256-GCM encryption for user API keys (BYOK) |
| `lib/logger.ts` | Pino structured logging with deployment helpers |
| `lib/rate-limit.ts` | Upstash Redis rate limiting |
| `lib/cache.ts` | LRU cache with TTL for frequently accessed data |
| `lib/prisma.ts` | Prisma client singleton with graceful shutdown |
| `lib/constants.ts` | Centralized constants (credit costs, limits) |
| `worker.ts` | Nexus runtime Nexus engine for background tasks |

## Customization

After running setup, customize for your product:

### 1. Update `lib/constants.ts`
Add product-specific constants:
```typescript
export const CREDIT_COSTS = {
  YOUR_ACTION: 5,
  // ...
} as const;
```

### 2. Update `prisma/schema.prisma`
Add product-specific models:
```prisma
model YourModel {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  // ...
}
```

### 3. Update `AGENTS.md`
Add product-specific rules in section 6.

### 4. Add MCP tools
Create MCP endpoints in `app/api/mcp/` for cross-product communication.

## Environment Variables

Required in `.env.local`:

```bash
# Database
DATABASE_URL="postgresql://..."

# Auth
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="..."  # Generated by setup

# BYOK Encryption
BYOK_ENCRYPTION_KEY="..."  # Generated by setup

# OAuth (optional)
GITHUB_CLIENT_ID=""
GITHUB_CLIENT_SECRET=""
GOOGLE_CLIENT_ID=""
GOOGLE_CLIENT_SECRET=""

# Upstash (optional)
UPSTASH_REDIS_REST_URL=""
UPSTASH_REDIS_REST_TOKEN=""
```
